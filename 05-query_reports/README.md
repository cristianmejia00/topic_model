# 05-query_reports

This folder contains the step-05 root reporting pipeline.

## S3 layout used by step 05

- Results root:
  - `s3://openalex-results/snapshot_{SNAPSHOT}/queries/{QUERY}/`
- Network root:
  - `s3://openalex-results/snapshot_{SNAPSHOT}/queries/{QUERY}/network/`
- Clustering/report root (all step-05 generated report datasets):
  - `s3://openalex-results/snapshot_{SNAPSHOT}/queries/{QUERY}/network/clustering/`

All report outputs generated by scripts in this folder are written under the
clustering/report root above.

Exception:

- Athena query-result staging is unchanged:
  - `s3://openalex-outputs/athena-staging/`

## Scripts in this folder

- `run_root_pipeline.py`
  - Entry point for root scripts in this folder.
- `root_common_config.py`
  - Shared env/path resolution and overwrite guards.
- `create_glue_catalog.py`
  - Creates/refreshes Glue database and crawler over step-05 source prefixes.
- `create_athena_reports.py`
  - Builds Athena tables in order:
    `article_report`, `cluster_report_micro`, `cluster_report_meso`, `cluster_report_macro`.
- `audit_athena_hierarchy.py`
  - Audits hierarchy consistency from `article_report`.
- `cluster_bertopic.py`
  - Generates BERTopic keywords, embeddings, and shared 2D coordinates.
- `create_macro_color_palette.py`
  - Creates deterministic macro color palette at `cluster_color_macro/`.
- `name_macro_clusters.py`
  - Creates macro cluster names at `cluster_name_macro/`.
- `migrate_meso_as_micro.py`
  - Optional migration that promotes `meso_cluster` to effective `micro_cluster`.
- `macro_palette.py`
  - Shared deterministic color helpers and palette loader.

## Required runtime context

`run_root_pipeline.py` requires:

- `--snapshot` (for S3 path template)
- `--query` (for S3 path template)

Glue/Athena database naming in step 05 is derived automatically as:

- `snapshot_{SNAPSHOT}-{QUERY}`

These are exported to scripts as environment variables:

- `TOPIC_MODEL_DATABASE`
- `TOPIC_MODEL_SNAPSHOT`
- `TOPIC_MODEL_QUERY`

## Typical execution order (step 05)

1. `create_glue_catalog.py`
2. Optional: `migrate_meso_as_micro.py`
3. `run_root_pipeline.py --step athena_reports`
4. `run_root_pipeline.py --step audit_hierarchy`
5. `run_root_pipeline.py --step bertopic`
6. Optional: `run_root_pipeline.py --step macro_colors`
7. Optional: `run_root_pipeline.py --step macro_names`

## Quick start

```bash
source .venv/bin/activate

# 1) Bootstrap Glue catalog over query outputs
python create_glue_catalog.py \
  --snapshot 2026-06-26 \
  --query q20260629 \
  --crawler-role AWSGlueServiceRole-nodes_crawler

# 2) Optional migration for canonical source table
python run_root_pipeline.py \
  --snapshot 2026-06-26 \
  --query q20260629 \
  --version version3 \
  --step meso_as_micro

# 3) Athena reports
python run_root_pipeline.py \
  --snapshot 2026-06-26 \
  --query q20260629 \
  --step athena_reports \
  --force

# 4) Audit
python run_root_pipeline.py \
  --snapshot 2026-06-26 \
  --query q20260629 \
  --step audit_hierarchy

# 5) BERTopic
python run_root_pipeline.py \
  --snapshot 2026-06-26 \
  --query q20260629 \
  --step bertopic \
  --force

# 6) Optional macro assets
python run_root_pipeline.py \
  --snapshot 2026-06-26 \
  --query q20260629 \
  --step macro_colors \
  --step macro_names \
  --force
```
